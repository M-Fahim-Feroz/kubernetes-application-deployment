# Useful Kubernetes Commands

This document provides a cheat sheet for verifying, troubleshooting, and managing the Kubernetes deployment.

## Cluster Info
```bash
# Get cluster info
kubectl cluster-info

# List all nodes
kubectl get nodes
```

## Pod & Deployment Management
```bash
# Watch pods spin up
kubectl get pods -n node-app-namespace -w

# Check deployment status
kubectl rollout status deployment/node-app -n node-app-namespace

# Describe a specific pod to troubleshoot errors (CrashLoopBackOff, ImagePullBackOff, etc.)
kubectl describe pod <pod-name> -n node-app-namespace

# View logs for a pod
kubectl logs <pod-name> -n node-app-namespace

# Port forward to a pod for local testing without Ingress
kubectl port-forward <pod-name> 3000:3000 -n node-app-namespace
```

## Networking (Services & Ingress)
```bash
# List services
kubectl get svc -n node-app-namespace

# Describe the service to ensure endpoints are registered
kubectl describe svc node-app-service -n node-app-namespace

# View Ingress resources
kubectl get ingress -n node-app-namespace

# Describe Ingress to check routing rules
kubectl describe ingress node-app-ingress -n node-app-namespace
```

## ConfigMaps and Secrets
```bash
# View the ConfigMap
kubectl get configmap node-app-config -n node-app-namespace -o yaml

# View the Secret (encoded)
kubectl get secret node-app-secret -n node-app-namespace -o yaml
```

## Helm Specific Commands
```bash
# List installed releases
helm ls -n node-app-namespace

# Get the manifest generated by the release
helm get manifest node-app -n node-app-namespace

# Upgrade a release after modifying values
helm upgrade node-app helm/node-app/ -n node-app-namespace
```
